# 전처리 요약 문서

## 1. 문서 목적
이 문서는 연인 갈등 상황 대응을 위한 감정 기반 RAG 챗봇 프로젝트에서 사용되는 전처리 결과물을 정리하고, 각 데이터가 어떤 목적을 위해 사용되는지 설명하기 위한 문서이다.

본 프로젝트의 전처리 목표는 단순 정제가 아니라, 핵심 기능별로 사용할 수 있도록 데이터를 역할별로 분리하는 데 있다.

핵심 기능은 다음과 같다.

1. 감정 및 갈등 위험도 분석
2. 유사 사례 검색 기반 답변 추천
3. 피해야 할 표현 및 대체 표현 제안

---

## 2. 전처리 결과물 구성

전처리 결과물은 총 4개 파일이며, 크게 두 축으로 나뉜다.

### 감정/위험도 분석용
- `continuous_dialogue_dialogue.csv`
- `continuous_dialogue_utterance.csv`

### RAG 검색/답변 추천용
- `rag_documents.csv`
- `response_pairs.csv`

---

## 3. 전처리 방식 요약

### 3-1. 연속 대화 데이터 전처리
연속 대화 데이터는 초기 전처리와 감정 라벨 정리를 분리하던 방식에서, 하나의 통합 전처리 스크립트로 정리되었다.

현재는 원본 엑셀 파일을 읽은 뒤
- 기본 구조 정리
- 대화 그룹 번호 생성
- 발화 단위 데이터 생성
- 감정 문자열 정리
- 감정 오타 수정
- 정상 감정만 유지
- turn_index 재생성
- 대화 단위 데이터 재생성
순서로 한 번에 처리하여 최종 CSV 2개를 생성한다.

즉, 감정 분석용 데이터는 중간 산출물을 따로 관리하지 않고 최종 전처리 파일 기준으로 사용한다.

### 3-2. 공감형 대화 데이터 전처리
공감형 대화 데이터는 연인 관계 대화만 추출한 뒤,
- RAG 검색용 문서 테이블
- 문맥-응답 pair 테이블
두 종류로 정리하였다.

또한 실제 협업과 앱 연결 편의를 위해 일부 결측값은 기준값으로 통일해 최종 사용 파일로 저장하였다.

---

## 4. 파일별 전처리 목적

### 4-1. continuous_dialogue_dialogue.csv
연속 대화 데이터를 대화 단위로 정리한 파일이다.

목적:
- 대화 전체 분위기 파악
- 감정 흐름 확인
- 갈등 위험도 보조 판단

구성:
- 대화 전체 텍스트
- 감정 시퀀스
- 발화 수

---

## 4-2. continuous_dialogue_utterance.csv
연속 대화 데이터를 발화 단위로 정리한 파일이다.

목적:
- 발화 단위 감정 분석
- 감정 분류 로직 설계
- 위험 표현 탐색

구성:
- 대화 ID
- 발화 순서
- 발화 텍스트
- 감정 라벨

---

## 4-3. rag_documents.csv
공감형 대화 데이터에서 연인 관계 대화를 추출하고, RAG 검색용 문서 형태로 정리한 파일이다.

목적:
- 유사 사례 검색
- 상황 기반 문서 검색
- 추천 답변 생성의 근거 문서 제공

구성:
- 상황 요약
- 화자 감정
- 청자 반응 정보
- 전체 대화
- 공감 태그
- 위험도
- 감정 변화
- 종료 여부

---

## 4-4. response_pairs.csv
공감형 대화에서 응답 직전 문맥과 실제 응답을 짝지어 정리한 파일이다.

목적:
- 추천 답변 예시 생성
- 문맥-응답 패턴 분석
- 공감 반응 참고

구성:
- 상황 요약
- 화자 감정
- 응답 직전 문맥
- 실제 청자 응답
- 공감 유형
- 종료 여부

## 6. 결측치 처리 기준

프로젝트는 실제 앱 연결과 팀 협업이 중요하므로, 일부 결측치는 삭제보다 의미 있는 기본값으로 채우는 방향을 사용했다.

### rag_documents.csv
- `final_speaker_change_emotion`의 기존 빈값 → `unknown`

### response_pairs.csv
- `listener_empathy`의 기존 빈값 → `미분류`

현재 저장된 `rag_documents.csv`와 `response_pairs.csv`는 위 기준이 반영된 최종 사용 파일이다.

## 7. 주의할 점

### 7-1. 파일 역할을 섞지 않는다
- 연속 대화 기반 2개 파일은 감정/위험도 분석 중심
- 공감형 대화 기반 2개 파일은 RAG 검색/답변 추천 중심

### 7-2. 종료 여부 컬럼 이름이 다르다
- `rag_documents.csv`는 `terminated`
- `response_pairs.csv`는 `terminate`

의미는 모두 대화 종료 여부이므로 문서상에서 동일 의미로 이해하면 된다.

### 7-3. 공감 관련 컬럼이 서로 다르다
- `listener_behavior`: 청자 반응 방식 요약
- `listener_empathy_tags`: 문서 단위 공감 태그
- `listener_empathy`: 응답 단위 공감 태그

### 7-4. 감정 분석용 연속 대화 데이터는 최종 전처리 파일 기준으로 사용한다
중간 산출물보다 아래 두 파일을 기준으로 개발을 진행한다.
- `continuous_dialogue_utterance.csv`
- `continuous_dialogue_dialogue.csv`

---

## 8. 최종 정리

1. 감정 분석 담당은 연속 대화 파일 2개를 기준으로 분석 진행
2. RAG 담당은 `rag_documents.csv`를 기준으로 유사 사례 검색 구조 설계
3. 답변 추천 담당은 `response_pairs.csv`를 기준으로 응답 예시 활용
4. UI/통합 담당은 최종 파일 4개를 기준으로 기능 연결 진행
